# üóÇÔ∏è Blob Resources

This guide shows how to upload and download blobs (binary files) using the SILstat‚Ñ¢ API. Blobs may include documents, reports, images, or any other binary data.

---

## üì¶ Uploading A Blob

To upload a file (blob), you must first create a `BlobResource` to associate the upload, its `Id` is used to perform the actual upload.

### ‚úÖ Step 1: Create a BlobResource

Send a `POST` request to `https://api.silstat.exsilentia.com/api/blobresources` to create the `BlobResource` to associate with the uploaded blob.

<Tabs>
<TabItem value="powershell" label="Powershell">

```powershell
$headers = @{
  Authorization = "Bearer <YOUR_ACCESS_TOKEN>"
  "Content-Type" = "application/json"
  # SelectedOrganizationId = "YOUR_SELECTED_ORGANIZATION" # Optional
}

# Define the BlobResource payload
$containerBody = @{
        "Uri": "<LOCATION_OF_FILE>",
        "AutomaticRevision": true,
        "MIMEType": "image/svg+xml",
        "MIMECategory": 2,
        "Extension": "svg",
        "Names": [
            {
                "Text": "Name of Blob",
                "LanguageISO639_2": "eng"
            }
        ],
        "AutogeneratedNames": true,
        "Descriptions": [
            {
                "Text": "Description of Blob",
                "LanguageISO639_2": "eng"
            }
        ],
        "AutogeneratedDescriptions": true,
        "Id": "123e4567-e89b-12d3-a456-426614174000",
    } | ConvertTo-Json -Depth 3

# Send the request to create the BlobResource
$response = Invoke-RestMethod -Uri "https://api.silstat.exsilentia.com/api/blobresources" `
  -Method Post -Headers $headers -Body $containerBody

```
</TabItem>

```bash
            {

```

### ‚úÖ Step 2: Upload Blob

After successfully creating a `BlobResource`, you can now upload a binary file (e.g., PDF, image, document) and associate it with the blob resource using its `Id`.

The blob will be stored in the system and linked to the specified `BlobResource` for future access, downloads, or tracking.

> üí° **Note:**  
> This example uses `multipart/form-data` to upload the file, which is the most common approach when sending binary data with metadata in PowerShell scripts.  
>
> However, the API also supports other methods of upload such as `application/octet-stream`.


<Tabs>
<TabItem value="powershell" label="Powershell">

```powershell
$headers = @{
  Authorization = "Bearer <YOUR_ACCESS_TOKEN>"
  # SelectedOrganizationId = "YOUR_SELECTED_ORGANIZATION" # Optional
}

# Define the file to upload
$filePath = "./yourfile.pdf"

# Prepare the multipart form data
$multipartForm = @{
  file = Get-Item $filePath
}

# Send the request to upload the file
$response = Invoke-RestMethod -Uri "https://api.silstat.exsilentia.com/api/blobs/upload/<BLOB_RESOURCE_ID_FROM_STEP_1>" `
  -Method Post -Headers $headers -Form $multipartForm

```
</TabItem>

<TabItem value="bash" label="CURL">

```bash
curl -X POST https://api.silstat.exsilentia.com/api/blobs/upload/<BLOB_RESOURCE_ID_FROM_STEP_1> \
  -H "Authorization: Bearer <YOUR_ACCESS_TOKEN>" \
  -F "file=@./yourfile.pdf" 
```
</TabItem>
</Tabs>

> üìé **Note:**  
> If the upload is successful, the response will include metadata for the uploaded blob, including a unique `Id` and `versionTag` that you should store for future retrieval or reference.  
>  
> In case of errors (e.g. invalid file, missing BlobResource ID, or authentication issues), the API will return a descriptive error message with an appropriate HTTP status code such as `400 Bad Request` or `401 Unauthorized`.

## üì• Downloading a Blob

You can download a blob using either the `Id` (returned in the response when the file was originally uploaded) to get the latest version, or by specifying both the `Id` and a `versionTag` to get a specific version.

The blob is retrieved via a `GET` request and streamed back as binary content. You can use the `--output` option in `curl` to save it locally.

### üîΩ 1. Download by `Id` (Latest Version)

<Tabs>
<TabItem value="powershell" label="Powershell">

```powershell
$headers = @{
  Authorization = "Bearer <YOUR_ACCESS_TOKEN>"
  # SelectedOrganizationId = "YOUR_SELECTED_ORGANIZATION" # Optional
}

$blobId = "<BLOB_ID>"
$outputFile = "downloaded-file.pdf"

Invoke-RestMethod -Uri "https://api.silstat.exsilentia.com/api/blobs/$blobId" `
  -Method Get -Headers $headers `
  -OutFile $outputFile

```
</TabItem>

<TabItem value="bash" label="CURL">

```bash
curl -X GET https://api.silstat.exsilentia.com/api/blobs/<BLOB_ID> \
  -H "Authorization: Bearer <YOUR_ACCESS_TOKEN>" \
  --output downloaded-file.pdf
```
</TabItem>
</Tabs>

---

### üîΩ 2. Download by `Id` and `versionTag` (Specific Version)

<Tabs>
<TabItem value="powershell" label="Powershell">

```powershell
$headers = @{
  Authorization = "Bearer <YOUR_ACCESS_TOKEN>"
  # SelectedOrganizationId = "YOUR_SELECTED_ORGANIZATION" # Optional
}

$blobId = "<BLOB_ID>"
$versionTag = "<VERSION_TAG>"
$outputFile = "downloaded-file-v2.pdf"

Invoke-RestMethod -Uri "https://api.silstat.exsilentia.com/api/blobs/$blobId/$versionTag" `
  -Method Get -Headers $headers `
  -OutFile $outputFile

```
</TabItem>

<TabItem value="bash" label="CURL">

```bash
curl -X GET https://api.silstat.exsilentia.com/api/blobs/<BLOB_ID>/<VERSION_TAG> \
  -H "Authorization: Bearer <YOUR_ACCESS_TOKEN>" \
  --output downloaded-file-v2.pdf
```
</TabItem>
</Tabs>

> üìé **Note:**  
> A successful download request will return the raw file content in the response body, along with key metadata in the response headers.
